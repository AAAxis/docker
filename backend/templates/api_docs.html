Theholylabs API Documentation

Welcome to the Theholylabs API documentation. This API provides various services including OpenAI proxy, file uploads, authentication, and more.

Base URL
https://api.theholylabs.com

Authentication

Most endpoints are public and don't require authentication. Some endpoints may have built-in security measures.

OpenAI Proxy

POST /api/openai

Proxy requests to OpenAI's API with built-in authentication.

Request Body:
{
  "model": "gpt-3.5-turbo",
  "messages": [
    {
      "role": "user",
      "content": "Hello, world!"
    }
  ],
  "max_tokens": 100
}
Response:
{
  "choices": [
    {
      "message": {
        "role": "assistant",
        "content": "Hello! How can I help you today?"
      }
    }
  ]
}

File Upload

POST /upload

Upload files and get a direct URL to access them.

Request (multipart/form-data):
file: [binary file data]
Response:
{
  "message": "File uploaded successfully",
  "file_url": "https://api.theholylabs.com/uploads/filename.ext"
}
GET /uploads/{filename}

Access uploaded files directly.

Resume Upload

POST /resume

Submit resume with personal information. Automatically sends to HR via Telegram.

Request (multipart/form-data):
name: "John Doe"
email: "john@example.com"
jobTitle: "Software Developer"
file: [resume file]
Response:
{
  "message": "File uploaded successfully and sent to Telegram",
  "file_url": "https://api.theholylabs.com/uploads/resume.pdf"
}

Payment Processing

POST /create-payment-intent

Create a Stripe payment intent for processing payments.

Request Body:
{
  "amount": 1000,
  "currency": "usd"
}
Response:
{
  "clientSecret": "pi_1234567890_secret_xyz"
}
POST /create-checkout-session

Create a Stripe checkout session for hosted payment page.

Request Body:
{
  "order": "ORDER123",
  "email": "customer@example.com",
  "total": "29.99",
  "name": "John Doe"
}
Response:
{
  "sessionUrl": "https://checkout.stripe.com/pay/cs_...",
  "status": "success"
}

Contact

GET /subscribe

Send contact messages with automatic email confirmation and Telegram notification.

Query Parameters:
name: "John Doe"
email: "john@example.com"
text: "Your message here"
zipcode: "12345" (optional)
Response:

Returns HTML thank you page or JSON success message.

Global Authentication

GET /global_auth

Send email verification codes for authentication purposes.

Query Parameters:
email: "user@example.com"
Response:
{
  "message": "Verification code sent successfully",
  "verification_code": "123456"
}

Error Handling

All endpoints return appropriate HTTP status codes and error messages:

Status Code Description
200 Success
400 Bad Request - Invalid parameters
500 Internal Server Error
